Quick Start

Last updated: June 22nd

LineCMS is a Content Management System built using a Flask framework (Python 3.*), LineCMS Provides Custom Themes with Jinja Templating engine, Shop functionality with Stripe's Payment Gateway and a blogging system

Download

Download the latet version of LineCMS

Download LineCMS

Installation

Download requirements

LineCMS requires some python dependency,

Install requirements using pip (LineCMS Requires Python >= 3.x.x)

Navigate to LineCMS folder, Console:

pip install -r requirements.txt Install all required LineCMS Dependencies

Run in Debug mode (Development)

Tweaks and test the CMS

Navigate to LineCMS folder, Console:

export FLASK_DEBUG=1 Enables Debug mode

flask run Run the CMS

Install Credentials

Once your server is running you can navigate to

http://your-website-link/install to add your Dashboard Credentials and redirected you to Dashboard once the installation is success.

Templates

Templates and resources files are split into 2 folders,
static is where your resources (JS/HTML/Images etc..)
static/SYSTEM/custom-theme/
templates folder is where your HTML files should be
templates/SYSTEM/custom-theme/

Folder name should have the same name and a Theme.py where the theme information resides

To create your own theme folder you can copy the whole 'blank-theme' inside the LineCMS, update it and compress into .zip file then move the static/templates to its folders or simply Upload the zip file using dashboard

Linking resources

Linking CSS,JS and other files into your HTML files


Use of static_link(from='custom-theme',get='style.css') template function, more detailed information about this function below.
<!DOCTYPE html> 
<html lang="en">            {% raw %}
<link rel="stylesheet" href="{{ static_link('custom-theme','style.css') }}">
                                            {% endraw %}

Template functions

Functions for your HTML files

Admin button

admin_button() Shows a hyperlink button to Dashboard if the 'logged' session exists (Logged in), returns string (HTML button)

Resource loader

link_for(theme='my-template',path='file.js') Loads the 'file.js' from 'my-template' in /static/SYSTEM/ folder 


    Args:
        theme (str): theme name
        path (str): file name or a folder path to a file

    Returns:
        string: url of a file

Load Products

load_products(counts=12,category="tech") Shows '12' products under 'tech' category


    Args:
        single (str, optional): product_id. Defaults to False.
        search (str, required if single = False): title/description search. Defaults to None.
        category (str, required if single = False): categorize search_. Defaults to None.
        offset (int, required if single = False): offset search. Defaults to 0.
        per_page (int, required if single = False): per_page for navigation. Defaults to 3.

    Returns:
        tuple: products

Load Blogs

load_blogs(counts=[0,5]) Load blogs with offset [0,5]


    Args:
        count (int [offset,offset], default): uses SQLite Limit. Defaults to 5.
        single (str, optional): blog route, loads single blog post. Defaults to False.

    Returns:
        tuple: blogs

Set Attribute

setattribute('product',my_product)
Purpose of setattribute() is to assign an attribute name base on column index from database,
instead of retrieving a 'category' value using product[3] you can use product.category. It's good to include setattribute() inside a Jinja's for loop.
example below with using 'product' as first argument to let LineCMS that it's a product object,
followed by iterable my_product object from database,
available keys: 'product','blog','site','mod','orders' 

{% raw %}

    {% for my_products in load_products(counts=[0,3]) %}            // jinja for-loop using load_products()
      {% set product = setattribute('product', my_products) %}       // create a variable product contains setattribute()
      
      <p>This is {{ product.title }} and for sale for {{ product.price }}</p>

    {% endfor %}

{% endraw %}

Without 'setattribute()' function you will have to use indexing to get a value
e.g to get 'category' value you have to use my_product[3] instead of my_product.category

Email templates (Enabled SMTP Required)

You will need to set-up your SMTP Settings enable to use the Templates below which will be sent to the customer's email

SMTP settings can be updated thru Dashboard > Products > Settings and look for the Email Settings

Email templates can be editted from Dashboard > Products > Settings

Fulfilled Template: Is the template used when a Customer order has been Fulfilled (Completed)

    {% raw %}
  • {{ COMPANYNAME }}: Loads the Name of your website
  • {{ COMPANYNUMBER }}: Loads the Phone Number of your website
  • {{ COMPANYEMAIL }}: Loads the Email of your website
  • {{ TRACKINGLINK }}: Loads the Tracking link you supplied (Will show if not empty)
  • {{ ORDERNUMBER }}: Loads a unique system generated order number
  • {{ ORDERTOTAL }}: Loads the amount of orders (USD) from Stripe
  • {{ ORDERDATE }}: Loads the creation date of the order
  • {{ CUSTOMERNAME }}: Loads the customer name

Placed Template: Is the template used when a Customer payment has been placed

  • {{ COMPANYNAME }}: Loads the Name of your website
  • {{ COMPANYNUMBER }}: Loads the Phone Number of your website
  • {{ COMPANYEMAIL }}: Loads the Email of your website
  • {{ ORDERTOTAL }}: Loads the amount of orders (USD) from Stripe
  • {{ ORDERDATE }}: Loads the creation date of the order
  • {{ CUSTOMERNAME }}: Loads the customer name
{% endraw %}
